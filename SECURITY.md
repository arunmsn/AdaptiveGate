# Security Policy

## Reporting a Vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities by emailing the maintainers listed in [MAINTAINERS.md](MAINTAINERS.md). Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigations

You will receive an acknowledgement within 48 hours and a status update within 7 days.

## Supply Chain

- All releases are signed with [cosign](https://github.com/sigstore/cosign)
- Every release includes an SBOM generated by [syft](https://github.com/anchore/syft)
- Dependencies are kept minimal by design; see `go.mod`
- `govulncheck` runs on every PR via CI
